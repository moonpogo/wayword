# PROMPT_ARCHITECTURE_V1

## 1. Purpose
Wayword prompts are not content. They are entry conditions for language.

This document defines the prompt architecture as a practical doctrine for editorial decisions, system behavior, and measurement boundaries.

## 2. Core Philosophy
Entering language is the movement from hesitation or diffuse attention into live verbal action.

The first job of a prompt is not depth, novelty, or insight. The first job is linguistic ignition.

Key principle: Creative latency is the antithesis of Wayword.

## 3. Evidence-Informed Stance
Wayword adopts an evidence-informed posture with clear scope:

- Writing initiation is constrained by planning load, working-memory demand, self-monitoring, and evaluation pressure.
- Low-stakes writing conditions support entry by reducing inhibitory overhead.
- Prompts should function as language acts, not topic assignments.
- Philosophy defines architectural intent; empirical research constrains claims, scope, and confidence.

Wayword does not over-claim therapeutic, diagnostic, or universal outcomes from prompt interaction.

## 4. Layer Model

### Layer 1 - Entry
Job: cross from hesitation into first language.

Traits:
- deictic anchoring
- naming
- nearby perception
- concrete footholds
- small linguistic acts
- low-stakes play

Failure modes:
- latency
- banality
- school tone
- emotional extraction

### Layer 2 - Torsion
Job: sustain motion by adding relation.

Traits:
- address
- reply
- contrast
- before/after
- almost/except
- refusal
- re-description
- another angle

Failure modes:
- workshop drama
- cleverness
- forced emotionality
- therapy drift

### Layer 3 - Resonance
Job: hold productive impasse.

Traits:
- paradox
- silence
- translation
- contradiction
- absence
- impossible-but-workable frames
- symbolic pressure
- unresolved return

Failure modes:
- fake-Zen
- mysticism cosplay
- decorative obscurity
- pseudo-philosophy

## 5. Progressive Architecture
The system should be progressive, not chooser-heavy.

Users should not be required to self-diagnose whether they need depth.

Default behavior: start with the lightest viable bridge, then intensify only after language is already moving or when the user explicitly opts in.

## 6. Current Layer 1 Foundation Set
- Describe the closest thing near you. Don’t overthink it.
- Pick a sound nearby and follow it.
- Write about something that looks better from far away.
- If today had a color palette, what’s in it?
- Pick an object in the room and make it seem mysterious.
- Start with something ordinary and make it worse.
- Write about a small inconvenience like it altered history.
- Take something annoying seriously.
- What’s the most forgettable part of your day so far?
- Write about a moment that felt longer than it was.
- A sentence you overheard. Invent the rest.
- If white noise could talk, what would it say?
- Are you more of a window or a door today?
- Pick a word to wear for the day.
- A sentence you can’t finish yet. Write it anyway.
- Describe something that looks different upside down.
- Continue a conversation that never happened.
- Explain something badly on purpose.
- Try a word you’ve never used before.
- Continue from: “That’s not what happened.”
- Write one true sentence and one false one. Don’t say which is which.
- Give a brief history of something that has no history.
- Name the feeling you don’t have a word for yet.
- Describe something twice. Make the second description contradict the first.
- Try selling something in the room to someone who doesn’t want it.
- Take the last word you said out loud. Write from it.
- Start with “Technically…”
- An appliance in the room has a complaint. What is it?
- Start with the wrong word on purpose.
- Write the worst possible opening sentence. Then keep going.

## 7. Anti-Patterns
The following are explicitly disallowed in Wayword prompt architecture:

- therapy language
- emotional extraction
- school assignment tone
- gamification
- streaks/badges/XP
- fake profundity
- pseudo-Zen
- AI trying to sound deep
- covert profiling
- mental-health inference
- unnecessary data capture
- prompts that require users to understand a philosophy before writing

## 8. Measurement Principles
Primary Layer 1 signal:
- time to first token

Also consider:
- first-line completion
- abandonment before typing
- reroll frequency
- qualitative trust feedback

Retention is not the sole truth source.

Some tactics are normatively disallowed even when they improve engagement.

## 9. Prompt Lifecycle
Prompts are curated, finite, and pruneable assets.

Wayword should prefer:
- smaller stronger sets
- seasonal rotation
- deliberate replacement
- editorial review

over infinite prompt generation.

## 10. Seasonal Overlay
Seasonal prompts should act as subtle cognitive posture shifts, not content drops or decorative themes.

Seasonality should not be loudly announced. Influence should emerge through prompt weighting and observatory residue over time.
