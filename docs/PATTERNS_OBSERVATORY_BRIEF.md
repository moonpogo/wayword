# Patterns Observatory Brief

## Purpose
Patterns is not an analytics dashboard.
Patterns is Wayword's observatory and almanac for how writing changes over time.

System framing:
- Writing surface = ritual
- Mirror = encounter
- Patterns = observatory
- Almanac and archive = long memory

## WAYWORD SYSTEMS DOCTRINE
- Reveal forces, not objects.
- Perception before interpretation.
- Elegance is a mode of understanding.
- Build fields, not dashboards.
- Suggest before declaring.
- Design thresholds, not counts.
- Cultivate bearing, not productivity.
- Let time appear as seasons.
- Data must earn poetry.
- Build living systems, not solved ones.

## Product Principles
- Patterns should reveal recurrence, rhythm, drift, and seasonality.
- The current season should always be visible.
- Free should show the season the user is in.
- Archive and Almanac can preserve past seasons later.
- Avoid productivity, streaks, goals, gamification, and fake emotional diagnosis.
- Avoid dashboard and KPI grammar.
- Use data only where it earns the visual form.
- Beauty must clarify, not decorate.

## Instrument Model

### 1. Season
Purpose:
Shows the current writing season.

Data:
last 90 days of saved runs, run dates, active days, word volume.

Visual grammar:
season wheel, almanac ring, radial field, quiet marks, monthly arcs, Wayword O and lens resonance.

Current implementation:
Current Season seed exists as Season Wheel in Patterns.

### 2. Trace Field
Purpose:
Shows language recurrence and clustering.

Data:
repeated words, repeated phrases, possibly semantic neighborhoods later.

Visual grammar:
constellation, gravitational field, particles, sparse labels, clusters.

Notes:
This should replace generic repeated-word lists over time.

### 3. Pulse
Purpose:
Shows rhythm and cadence.

Data:
sentence length, punctuation, paragraph breaks, hesitation and qualification, compression and expansion.

Visual grammar:
waveform, seismograph, breath trace, pressure bands.

### 4. Drift Atlas
Purpose:
Shows long-horizon changes and returns.

Data:
motifs, themes, signal changes across seasons, recurring and fading and returning language.

Visual grammar:
sediment, strata, topographic bands, rivers of recurrence, field drift.

Notes:
This should probably wait for enough history and may belong to Almanac and archive.

## Layout Principles
- Desktop is comparative.
- Mobile is sequential.
- Desktop may eventually show instruments in a composed observatory grid.
- Mobile should show one instrument at a time or stacked previews.
- Never cram the whole observatory into mobile.
- Mobile instruments should feel like tactile specimens, not dashboards.

## Current Season Direction
- The current implementation is a seed, not the final form.
- The Season Wheel should evolve toward a richer, more tactile seasonal instrument.
- Avoid rectangular activity grids and GitHub contribution-chart grammar.
- Avoid making run counts the center of the section.
- The season name should orient the user.
- The artifact should carry the data presence.
- Text should explain less as the visual becomes stronger.

## What To Avoid
- GitHub contribution chart grammar
- Apple Fitness ring grammar
- KPI cards
- streaks
- productivity scoring
- faux therapy language
- over-labeled charts
- decorative data art that does not reveal anything
- heavy color coding
- dashboards pretending to be poetry

## Monetization Frame
Free:
Current Season, rolling 90 days, enough to observe the season the user is in.

Paid and Archive later:
Almanac, past seasons, annual cycle, motif returns, deeper strata, exports and yearbooks and prints.

"Give away wonder. Charge for depth."

## Near-Term Roadmap
1. Stabilize Current Season branch.
2. Dogfood Season Wheel on mobile and desktop.
3. Decide if wheel needs one more visual-density pass.
4. Next candidate instrument: Trace Field, using existing repeated-word data.
5. Later candidate: Pulse.
6. Delay Drift Atlas until enough longitudinal data exists.
