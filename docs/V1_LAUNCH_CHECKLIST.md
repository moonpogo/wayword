# Wayword V1 Launch Checklist

Use this checklist for the final manual launch-readiness pass.

Target duration: 20-30 minutes.

## 1. Core Desktop Loop

- [ ] Open Wayword on desktop in a clean browser tab.
- [ ] Confirm landing renders correctly.
- [ ] Click `Begin`.
- [ ] Confirm the app shell opens cleanly and the editor is immediately usable.
- [ ] Confirm a prompt is visible.
- [ ] Type a normal draft.
- [ ] Confirm submit becomes available only after text is present.
- [ ] Submit the run.
- [ ] Confirm Mirror appears.
- [ ] Confirm Mirror cards render as statement-only.
- [ ] Confirm no blank post-submit state appears.
- [ ] Confirm the next run can begin cleanly from the existing shell.

## 2. Recent Runs + Patterns

- [ ] Save at least one run.
- [ ] Open Recent Runs.
- [ ] Close Recent Runs.
- [ ] Reopen a saved run.
- [ ] Confirm the restored content/output matches the saved run.
- [ ] Save enough runs to unlock `Patterns`.
- [ ] Open `Patterns`.
- [ ] Close `Patterns`.
- [ ] Confirm no panel layering bug appears.
- [ ] Confirm no obvious overflow, clipping, or broken card layout appears.

## 3. Mobile Safari Pass

- [ ] Open Wayword in mobile Safari.
- [ ] Confirm landing renders correctly.
- [ ] Tap `Begin`.
- [ ] Confirm the editor is usable and focus behavior feels normal.
- [ ] Type and submit a normal draft.
- [ ] Confirm Mirror renders correctly on mobile.
- [ ] Open `Options`.
- [ ] Close `Options`.
- [ ] Open `Recent Runs` drawer.
- [ ] Close `Recent Runs` drawer.
- [ ] Open `Patterns` from the style tab once available.
- [ ] Close `Patterns`.
- [ ] Confirm the app returns cleanly to writing state.
- [ ] Confirm no stuck focus, trapped panel, or broken scroll state appears.

## 4. Persistence Refresh Pass

- [ ] With saved runs present, refresh the app.
- [ ] Confirm saved runs remain available after refresh.
- [ ] Confirm Recent Runs still opens correctly after refresh.
- [ ] Confirm Patterns still works after refresh when previously unlocked.
- [ ] Confirm the app can begin a clean new run after refresh.

## 5. Reroll Sanity

- [ ] Start a fresh run with an empty editor.
- [ ] Confirm reroll is available while the editor is still empty.
- [ ] Use reroll once.
- [ ] Start typing.
- [ ] Confirm reroll is no longer available once text is present.
- [ ] Confirm no reroll path remains after submit.

## 6. Trust / Copy Pass

- [ ] Confirm landing copy reads cleanly and matches the restrained V1 product stance.
- [ ] Confirm Mirror does not read like advice, diagnosis, or personality inference.
- [ ] Confirm no evidence controls, `Context`, `Hide`, debug text, counts, ratios, or internal scoring appear in the visible UI.
- [ ] Confirm Recent Runs and Patterns copy reads cleanly without obviously broken or duplicated text.

## 7. Launch Signoff

- [ ] Core desktop loop passed
- [ ] Recent Runs + Patterns passed
- [ ] Mobile Safari pass passed
- [ ] Persistence refresh pass passed
- [ ] Reroll sanity passed
- [ ] Trust/copy pass passed
